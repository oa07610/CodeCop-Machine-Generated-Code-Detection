# GPT-OSS 20B QLoRA Fine-tuning Approach

Machine-generated code detection using GPT-OSS 20B with QLoRA fine-tuning via Unsloth.

## Overview

This approach leverages a large 20B parameter decoder model with 4-bit quantization (QLoRA) for memory-efficient fine-tuning. The model is trained as a generative classifier, outputting "human" or "machine" as text.

## Files

- `peft_gpt-oss:20b.ipynb` - Complete training notebook (Google Colab compatible)
- `../../predictions/gpt_oss_predictions.csv` - Model predictions on test set

## Model Details

- **Base Model**: `unsloth/gpt-oss-20b`
- **Method**: QLoRA (4-bit quantization + LoRA adapters)
- **Max Sequence Length**: 1024 tokens
- **Framework**: Unsloth (2x faster training)
- **Task**: Generative classification → "human" or "machine"

```bash
pip install -r requirements.txt
```

## Training (Google Colab)

1. Open `peft_gpt-oss:20b.ipynb` in Google Colab
2. Select a GPU runtime (Runtime → Change runtime type → T4 GPU or better)
3. Run all cells in order
4. The notebook will:
   - Install dependencies (transformers, unsloth, trl, etc.)
   - Load the SemEval-2026 Task 13 dataset from HuggingFace
   - Fine-tune GPT-OSS 20B with QLoRA
   - Generate predictions on the test set

## Requirements

- **GPU**: T4 (minimum), A100 recommended for faster training
- **VRAM**: ~15GB with 4-bit quantization
- **Disk**: ~40GB for model weights

## Training Configuration

| Parameter | Value |
|-----------|-------|
| LoRA Rank | 16 |
| LoRA Alpha | 32 |
| Dropout | 0.05 |
| Quantization | 4-bit (NF4) |
| Max Seq Length | 1024 |

## Notes

- Uses Unsloth for 2x faster training on consumer GPUs
- Offloads embeddings to RAM to save GPU memory
- Compatible with HuggingFace TRL for SFT training
